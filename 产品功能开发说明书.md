# 产品功能开发说明书

## 1. 文档目标

本文件为**可直接交付开发团队实施的技术说明书**，覆盖： - 模块设计 -
数据结构 - 接口协议 - 状态机 - 异常规范 - 数据库SQL - 开发约束

------------------------------------------------------------------------

## 2. 系统架构

### 2.1 分层架构

-   UI层（PySide6）
-   应用层（流程控制）
-   服务层（OCR / AI / 配置 / 结果）
-   数据层（SQLite）

------------------------------------------------------------------------

## 3. 核心业务流程

    截图 → OCR → AI解析 → JSON编辑 → 入库（覆盖写）

------------------------------------------------------------------------

## 4. 模块设计

### 4.1 tray模块

职责： - 托盘初始化 - 菜单绑定

接口： - init_tray() - bind_events()

------------------------------------------------------------------------

### 4.2 capture模块

职责： - 截图逻辑 - 选区控制

接口： - start_capture() - cancel_capture() - get_image_path()

------------------------------------------------------------------------

### 4.3 ocr_service

接口：

    def run_ocr(image_path: str) -> str

请求： POST /api/ocr

返回：

    {
      "code": 100,
      "data": {"text": "xxx"}
    }

异常码： - OCR_001：连接失败 - OCR_002：返回异常 - OCR_003：空文本

------------------------------------------------------------------------

### 4.4 ai_service

接口：

    def run_ai(prompt: str, ocr_text: str) -> str

请求（OpenAI兼容）：

    POST /v1/chat/completions
    {
      "model": "xxx",
      "messages": [
        {"role": "system", "content": "只返回JSON"},
        {"role": "user", "content": "Prompt + OCR"}
      ]
    }

返回：

    {
      "choices":[{"message":{"content":"JSON字符串"}}]
    }

异常码： - AI_001：连接失败 - AI_002：认证失败 - AI_003：非JSON返回

------------------------------------------------------------------------

### 4.5 result_service

接口：

    def save_result(data)

逻辑： - 若存在 → UPDATE - 不存在 → INSERT

------------------------------------------------------------------------

## 5. 状态机

状态： - idle - capturing - preview - ocr_processing - ai_processing -
editing - saving

流转： idle → capturing → preview → ocr → ai → editing → saving → idle

------------------------------------------------------------------------

## 6. 数据库设计（SQL）

### capture_types

    CREATE TABLE capture_types (
      id INTEGER PRIMARY KEY,
      name TEXT UNIQUE,
      description TEXT,
      prompt_template TEXT,
      is_enabled INTEGER,
      created_at TEXT,
      updated_at TEXT
    );

### ai_providers

    CREATE TABLE ai_providers (
      id INTEGER PRIMARY KEY,
      name TEXT,
      api_base_url TEXT,
      api_key TEXT,
      is_enabled INTEGER,
      is_default INTEGER
    );

### ai_models

    CREATE TABLE ai_models (
      id INTEGER PRIMARY KEY,
      provider_id INTEGER,
      model_code TEXT,
      model_name TEXT,
      is_enabled INTEGER,
      is_default INTEGER
    );

### analysis_results

    CREATE TABLE analysis_results (
      id INTEGER PRIMARY KEY,
      result_date TEXT,
      capture_type_id INTEGER,
      image_path TEXT,
      ocr_text TEXT,
      ai_raw_response TEXT,
      final_json_text TEXT,
      created_at TEXT,
      updated_at TEXT,
      UNIQUE(result_date, capture_type_id)
    );

------------------------------------------------------------------------

### CaptureType编辑

-   description

-   is_enabled

### AI配置

-   provider.name

-   model_code
-   is_default

------------------------------------------------------------------------

## 8. 线程模型

采用： - QThreadPool + QRunnable

任务： - OCR任务 - AI任务

要求： - UI线程不可阻塞 - 使用signal返回结果

------------------------------------------------------------------------

## 9. JSON校验规则

-   必须为合法JSON
-   必须为对象（dict）
-   禁止空字符串

错误码： - JSON_001：解析失败

------------------------------------------------------------------------

## 10. 异常处理规范

统一结构：

    {
      "code": "AI_001",
      "message": "连接失败"
    }

------------------------------------------------------------------------

## 11. 开发规范

-   分层清晰
-   service不可依赖UI
-   数据访问统一DAO
-   所有IO操作需异常捕获

------------------------------------------------------------------------
